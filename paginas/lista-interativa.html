Lista Interativa

Construindo uma lista interativa com a biblioteca wd

22 de novembro de 2020

Apresentação

O propósito deste artigo é descrever a construção de uma lista interativa utilizando as ferramentas de script e de atributos disponibilizadas pela biblioteca wd.

O manual da biblioteca wd pode ser encontrado no link .

Elementos Básicos

Para demonstrar a construção da lista interativa, será utilizado como exemplo a seleção de Unidades Federativas - UF e respectivos Municípios.

Para evitar longas listas, as UF ficarão restritas à região sul do Brasil e apenas algumas cidades estarão disponíveis.

As informações ficarão armazenadas em arquivos JSON conforme abaixo:

Unidades Federativas - UF

json/PR.json


		
Santa Catarina - SC


		


		
json/SC.json


		
Rio Grande do Sul - RS


		


		
json/RS.json


		
Utilizando o Atributo


		
Para construção da lista interativa será utilizado o atributo data-wd-repeat. Esse atributo executa a repetição dos elementos filhos, definidos como modelo, substituindo os valores das palavras chaves ({{chave}}) pelos valores dos atributos definidos no arquivo JSON.


		
Nesta demostração será construído um navegador para definir a UF a ser exibida e uma lista simples para exibir os respectivos municípios. A lista precisará ter um identificador (id) para ser possível referenciá-la:


		
UF
		
		


		
Municípios

		
    

		
    h5UF/h5
nav/nav

h5Municípios/h5
ul id=municipios /ul
    

		
    Para carregar as listas, será necessário definir um elemento filho modelo, que servirá como molde para a construção de sua estrutura. Para fazer referências utilizando o atributo data-wd-repeat, usa-se chaves duplas com o nome do atributo do arquivo JSON. Por exemplo, caso queira fazer referência ao atributo nome do arquivo JSON, deveria referenciá-lo na forma {{nome}}.
    

		
    Os modelos poderiam ser construídos da seguinte forma:
    

		
    UF
    
		
    
			{{nome}}-{{sigla}}
		
    

		
    Municípios
    
		
      
			
    • {{nome}}
      • 
		
    

		
    h5UF/h5
nav
	span{{nome}}-{{sigla}}/span
/nav

h5Municípios/h5
ul id=municipios 
	li{{nome}}/li
/ul
    

		
    Inicialmente será construído o navegador de UF a partir do uso do atributo data-wd-repeat:
    

		
    UF
    
		
    
			{{nome}}-{{sigla}}
		
    

		
    Municípios
    
		
      
			
    • {{nome}}
      • 
		
    

		
    h5UF/h5
nav data-wd-repeat=get{json/UF.json}
	span{{nome}}-{{sigla}}/span
/nav

h5Municípios/h5
ul id=municipios1 
	li{{nome}}/li
/ul
    

		
    O próximo passo é definir a ação de carregar a lista das cidades ao clicar sobre uma UF no navegador. Para isso será utilizado o atributo data-wd-data. Esse atributo altera o valor do atributo HTML data e reage ao evento click.
     

		
    Adicionando o citado atributo ao elemento modelo do navegador:
    

		
    UF
    
		
    
			{{nome}}-{{sigla}}
		
    

		
    Municípios
    
		
      
			
    • {{nome}}
      • 
		
    

		
    h5UF/h5
nav data-wd-repeat=get{json/UF.json}
	span data-wd-data=wdRepeat{get{ {{municipios}} }}${#municipios2} {{nome}}-{{sigla}}/span
/nav

h5Municípios/h5
ul id=municipios2 
	li{{nome}}/li
/ul
    

		
    Observe que quando a página é carregada, antes de uma UF ser clicada no navegador, a lista fica exibindo o modelo ({{nome}}), o que é um pouco deselegante. Uma maneira de contornar tal situação seria utilizando o atributo data-wd-click sobre uma das opções do navegador. Esse atributo promove um clique automático após o carregamento da página.
    

		
    Mas para utilizá-lo aqui, deveríamos colocar a informação de qual seria a escolha padrão no arquivo JSON e referenciar o respetivo atributo no modelo. Por exemplo, poderíamos acrescentar o atributo inicial no arquivo JSON e deixá-lo com valor data-wd-click= para a UF inicial e em branco para as demais UF:
    

		
    -- Arquivo JSON --

[
	{
		sigla: PR,
		nome: Paraná,
		municipios: json/PR.json",
		inicial: data-wd-click=
	},
	{
		sigla: SC,
		nome: Santa Catarina,
		municipios: json/SC.json,
		inicial: 
	},
	...
]

-- HTML --

h5UF/h5
nav data-wd-repeat=get{json/UF.json}
	span data-wd-data=wdRepeat{get{ {{municipios}} }}${#municipios2} {{inicial}} {{nome}}-{{sigla}}/span
/nav
    

		
    Uma outra solução seria definir o atributo data-wd-repeat com a informação de um arquivo inexiste:
    

		
    UF
    
		
    
			{{nome}}-{{sigla}}
		
    

		
    Municípios
    
		
      
			
    • {{nome}}
      • 
		
    

		
    h5UF/h5
nav data-wd-repeat=get{json/UF.json}
	span data-wd-data=wdRepeat{get{ {{municipios}} }}${#municipios3}{{nome}}-{{sigla}}/span
/nav

h5Municípios/h5
ul id=municipios3 data-wd-repeat=get{null} 
	li{{nome}}/li
/ul
    

		
    Utilizando um Script
    

		
    A situação apresentada até aqui geralmente é encontrada em formulários, através da utilização de combos (select). Modificando o exemplo anterior para se adequar a um formulário, tem-se o seguinte:
    

		
    
			
			

			
			
		
    

		
    form class=wd-block
	labelUF/label
	select data-wd-repeat=get{json/UF.json} 
		option data-wd-data=wdRepeat{get{ {{municipios}} }}${#municipios4}{{nome}}-{{sigla}}/option
	/select

	labelMunicípios/label
	select id=municipios4 data-wd-repeat=get{null} 
		option{{nome}}/option
	/select
/form
    

		
    O formulário acima até que funciona, mas somente se os elementos option receberem um clique, pois é assim que o atributo data-wd-data funciona. Aa lista das cidades não será carregada se, por exemplo, mudarmos os valores de UF com o teclado.
    

		
    Para resolver isso, será utilizado um script que promoverá o carregamento dos municípios. Uma possível solução seria:
    

		
    function municipios() {
	var arquivo = wd$(#uf5).item().value;
	wd$(#municipios5).data({
		wdRepeat: get{+arquivo+}
	});
	return;
}

wd$(#uf5).handler({
	change: municipios
});

    
	
	
    Aplicando ao formulário:
    

		
    
			
			

			
			
		
    

		

		
    form class=wd-block
	labelUF/label
	select id=uf5 data-wd-repeat=get{json/UF.json} 
		option value={{municipios}} {{nome}}-{{sigla}}/option
	/select

	labelMunicípios/label
	select id=municipios5 data-wd-repeat=get{json/PR.json} 
		option{{nome}}/option
	/select
/form

script
function municipios() {
	var arquivo = wd$(#uf5).item().value;
	wd$(#municipios5).data({
		wdRepeat: get{+arquivo+}
	});
	return;
}

wd$(#uf5).handler({
	change: municipios
});
/script
    

		
    Forma Alternativa
    

		
    Em algumas situações não há jeito certo, existe aquele que não funciona e aquele que funciona, com mais ou menos erros. Portanto, métodos alternativos podem ser desenvolvidos sem precisar ficar preso a um único modelo.
    

		
    Para elaborar um formulário alternativo àquele acima apresentado, será utilizado o atributo data-wd-set, que define determinados atributos dos elementos, e o atributo data-wd-filter, que filtra os elementos de acordo com o texto pesquisado. O primeiro reage ao evento click e o segundo ao evento input.
    

		
    Segue abaixo o modelo alternativo para selecionar UF e Município sem utilizar nenhum script adicional. O código está longo devido a forma como foram distribuídos os atributos dos elementos HTML, pensados na organização e visualização. Clique no campo abaixo:
    

		
    

			

			
    
				
    {{nome}}-{{sigla}}
    
			
    

			

			
    

				
    

					

				
    


				
    

					
    {{nome}}
    
					
				
    
			
    

		
    

		
    form class=wd-block

	input
		type=text
		id=campoUF
		value=
		readonly=
		placeholder=Unidade Federativa
		data-wd-set=class{}${#listaUF}
	 /

	div
		id=listaUF
		class=wd-overlook
		data-wd-repeat=get{json/UF.json}
	
		div 
			class=wd-padding wd-border-bottom wd-upon-enter wd-bg-black
			data-wd-data=wdRepeat{get{ {{municipios}} }}${#listaCidades}
			data-wd-set=class{wd-overlook}${#listaUF}&value{{{sigla}}}${#campoUF}&value{}${#campoCidade}&class{wd-overlook}${#cidades}
		{{nome}}-{{sigla}}/div
	/div

	input
		type=text
		id=campoCidade
		value=
		readonly=
		placeholder=Município
		data-wd-set=class{}${#cidades}
	/

	div
		id=cidades
		class=wd-overlook
	

		div class=wd-block wd-padding wd-bg-black

			input
				type=search
				id=pesquisa
				placeholder=Pesquisar...
				data-wd-filter=show{}${#listaCidades}
			/

		/div


		div
			id=listaCidades
			data-wd-repeat=get{null}
		

			div
				class=wd-padding wd-border-bottom wd-upon-enter wd-bg-black
				data-wd-set=class{wd-overlook}${#cidades}&value{{{nome}}}${#campoCidade}
			{{nome}}/div
			
		/div
	/div

/form
