Lista Interativa

Construindo uma lista interativa com a biblioteca wd

20 de novembro de 2020

Apresentação

O propósito deste artigo é descrever a construção de uma lista interativa utilizando as ferramentas de script e de atributos disponibilizadas pela biblioteca wd.

O manual da biblioteca wd pode ser encontrado no link .

Elementos Básicos

Para demonstrar a construção da lista interativa, será utilizado como exemplo a seleção de Unidades Federativas - UF e respectivos Municípios.

Para evitar longas listas, as UF ficarão restritas à região sul do Brasil e apenas algumas cidades estarão disponíveis.

As informações ficarão armazenadas em arquivos JSON conforme abaixo:

Unidades Federativas - UF



		
Paraná - PR


		


		
json/PR.json


		
Santa Catarina - SC


		


		
json/SC.json


		
Rio Grande do Sul - RS


		


		
json/RS.json


		
Utilizando o Atributo


		
Para construção da lista interativa será utilizado o atributo data-wd-repeat. Esse atributo executa a repetição dos elementos filhos, definidos como modelo, substituindo os valores das palavras chaves ({{chave}}) pelos valores dos atributos definidos no arquivo JSON.


		
Nesta demostração será construído um navegador para definir a UF a ser exibida e uma lista simples para exibir os respectivos municípios. A lista precisará ter um identificador (id) para ser possível referenciá-la:


		
UF
		
		


		
Municípios

		
    

		
    h5UF/h5
nav/nav

h5Municípios/h5
ul id=municipios /ul
    

		
    Para carregar as listas, será necessário definir um elemento filho modelo, que servirá como molde para a construção de sua estrutura. Para fazer referências utilizando o atributo data-wd-repeat, usa-se chaves duplas com o nome do atributo do arquivo JSON. Por exemplo, caso queira fazer referência ao atributo nome do arquivo JSON, deveria referenciá-lo na forma {{nome}}.
    

		
    Os modelos poderiam ser construídos da seguinte forma:
    

		
    UF
    
		
    
			{{nome}}-{{sigla}}
		
    

		
    Municípios
    
		
      
			
    • {{nome}}
      • 
		
    

		
    h5UF/h5
nav
	span{{nome}}-{{sigla}}/span
/nav

h5Municípios/h5
ul id=municipios 
	li{{nome}}/li
/ul
    

		
    Inicialmente será construído o navegador de UF a partir do uso do atributo data-wd-repeat:
    

		
    UF
    
		
    
			{{nome}}-{{sigla}}
		
    

		
    Municípios
    
		
      
			
    • {{nome}}
      • 
		
    

		
    h5UF/h5
nav data-wd-repeat=get{json/UF.json}
	span{{nome}}-{{sigla}}/span
/nav

h5Municípios/h5
ul id=municipios1 
	li{{nome}}/li
/ul
    

		
    O próximo passo é definir a ação de carregar a lista das cidades ao clicar sobre uma UF no navegador. Para isso será utilizado o atributo data-wd-data. Esse atributo altera o valor do atributo HTML data e reage ao evento click.
     

		
    Adicionando o citado atributo ao elemento modelo do navegador:
    

		
    UF
    
		
    
			{{nome}}-{{sigla}}
		
    

		
    Municípios
    
		
      
			
    • {{nome}}
      • 
		
    

		
    h5UF/h5
nav data-wd-repeat=get{json/UF.json}
	span data-wd-data=wdRepeat{get{ {{municipios}} }}${#municipios2} {{nome}}-{{sigla}}/span
/nav

h5Municípios/h5
ul id=municipios2 
	li{{nome}}/li
/ul
    

		
    Observe que quando a página é carregada, antes de uma UF ser clicada no navegador, a lista fica exibindo o modelo ({{nome}}), o que é um pouco deselegante. Uma maneira de contornar tal situação seria utilizando o atributo data-wd-click sobre uma das opções do navegador. Esse atributo promove um clique automático após o carregamento da página.
    

		
    Mas para utilizá-lo aqui, deveríamos colocar a informação de qual seria a escolha padrão no arquivo JSON e referenciar o respetivo atributo no modelo. Por exemplo, poderíamos acrescentar o atributo inicial no arquivo JSON e deixá-lo com valor data-wd-click= para a UF inicial e em branco para as demais UF:
    

		
    -- Arquivo JSON --

[
	{
		sigla: PR,
		nome: Paraná,
		municipios: json/PR.json",
		inicial: data-wd-click=
	},
	{
		sigla: SC,
		nome: Santa Catarina,
		municipios: json/SC.json,
		inicial: 
	},
	...
]

-- HTML --

h5UF/h5
nav data-wd-repeat=get{json/UF.json}
	span data-wd-data=wdRepeat{get{ {{municipios}} }}${#municipios2} {{inicial}} {{nome}}-{{sigla}}/span
/nav
    

		
    Uma outra solução seria definir o atributo data-wd-repeat com a informação de um arquivo inexiste:
    

		
    UF
    
		
    
			{{nome}}-{{sigla}}
		
    

		
    Municípios
    
		
      
			
    • {{nome}}
      • 
		
    

		
    h5UF/h5
nav data-wd-repeat=get{json/UF.json}
	span data-wd-data=wdRepeat{get{ {{municipios}} }}${#municipios3}{{nome}}-{{sigla}}/span
/nav

h5Municípios/h5
ul id=municipios3 data-wd-repeat=get{X} 
	li{{nome}}/li
/ul
    















































	
	
	Para estabelecer a interação com o carrossel, será utilizado o atributo data-wd-action. Esse atributo executa a ação definida e reage ao evento click.

	
    Para acessar os slides específicos, será utilizada a ação tab, indicando o identificador do elemento (filho), e para acessar os elementos posterior ou anterior serão utilizadas as ações next e prev, respectivamente. Para essas últimas ações, o elemento pai também deverá receber um identificador, cujo seletor será referenciado na interação:
    

		
    
			
    Slide 1
    
			
    Slide 2
    
			
    Slide 3
    
			
    Slide 4
    
		
    

		
    
			
			1
			2
			3
			4
			
		
    

		
    div class=wd-b wd-size-xlarge wd-align-center id=slides1
	div style=height: 150px; id=slide1-1 class=wd-bg-blue  Slide 1/div
	div style=height: 150px; id=slide1-2 class=wd-bg-red   Slide 2/div
	div style=height: 150px; id=slide1-3 class=wd-bg-green Slide 3/div
	div style=height: 150px; id=slide1-4 class=wd-bg-black Slide 4/div
/div

nav
	span class=wd-icon-w data-wd-action=prev{#slides1}/span
	span data-wd-action=tab{#slide1-1}1/span
	span data-wd-action=tab{#slide1-2}2/span
	span data-wd-action=tab{#slide1-3}3/span
	span data-wd-action=tab{#slide1-4}4/span
	span class=wd-icon-e data-wd-action=next{#slides1}/span
/nav
    

		
    Note que todos os elementos são exibidos ao renderizar a página. Caso seja necessário que apenas um apareça, basta adionar o atributo data-wd-click ao botão que seleciona o elemento escolhido. Esse atributo promove um clique automático após o carregamento da página. Como exemplo, será definido como elemento inicial o slide2:
    

		
    
			
    Slide 1
    
			
    Slide 2
    
			
    Slide 3
    
			
    Slide 4
    
		
    

		
    
			
			1
			2
			3
			4
			
		
    

		
    div class=wd-b wd-size-xlarge wd-align-center id=slides2
	div style=height: 150px; id=slide2-1 class=wd-bg-blue  Slide 1/div
	div style=height: 150px; id=slide2-2 class=wd-bg-red   Slide 2/div
	div style=height: 150px; id=slide2-3 class=wd-bg-green Slide 3/div
	div style=height: 150px; id=slide2-4 class=wd-bg-black Slide 4/div
/div

nav
	span class=wd-icon-w data-wd-action=prev{#slides2}/span
	span data-wd-action=tab{#slide2-1}1/span
	span data-wd-action=tab{#slide2-2} data-wd-click= 2/span
	span data-wd-action=tab{#slide2-3}3/span
	span data-wd-action=tab{#slide2-4}4/span
	span class=wd-icon-e data-wd-action=next{#slides2}/span
/nav
    

		
    Exibição Automática
    

		
    A partir da construção anterior, com pequenas modificações, será implementado a transição automática dos elementos filhos.
    
		
		
    No lugar de construir um navegador para manipular a exibição dos elementos, apenas as ações next e prev permanecerão, entretanto, posicionadas dentro do container de slides, uma à direita e outra à esquerda, respectivamente.
    
		
		
    Para isso, um container externo deverá ser criado com posicionamento alterado para algo diferente do padrão (static), será escolhido o relative. Os botões com as ações serão inseridos dentro deste novo elemento pai através do objeto span e receberão os atributos de classe wd-place-.... O container com os slides também será adcionado ao novo elemento de nível superior. Os identificadores dos slides não serão mais necessários. Após as modificações necessárias, a visualização ficará da seguinte forma:
    

		
    
			
    
				
    Slide 1
    
				
    Slide 2
    
				
    Slide 3
    
				
    Slide 4
    
			
    
			
			
		
    

		
    div style=position: relative; class=wd-b wd-size-xlarge wd-align-center
	div id=slides3
		div style=height: 150px; class=wd-bg-blue  Slide 1/div
		div style=height: 150px; class=wd-bg-red   Slide 2/div
		div style=height: 150px; class=wd-bg-green Slide 3/div
		div style=height: 150px; class=wd-bg-black Slide 4/div
	/div
	span data-wd-action=next{#slides3} class=wd-place-e wd-padding wd-icon-right/span
	span data-wd-action=prev{#slides3} class=wd-place-w wd-padding wd-icon-left/span
/div
    

	
    Para implementar o andamento automático, será utilizado o atributo data-wd-slide. Esse atributo promove a alternância progressiva dos elementos filhos num certo intervalo de tempo representado, em milisegundos, pelo valor do atributo:
    

		
    
			
    
				
    Slide 1
    
				
    Slide 2
    
				
    Slide 3
    
				
    Slide 4
    
			
    
			
			
		
    

		
    div style=position: relative; class=wd-b wd-size-xlarge wd-align-center
	div id=slides3 data-wd-slide=4500 
		div style=height: 150px; class=wd-bg-blue  Slide 1/div
		div style=height: 150px; class=wd-bg-red   Slide 2/div
		div style=height: 150px; class=wd-bg-green Slide 3/div
		div style=height: 150px; class=wd-bg-black Slide 4/div
	/div
	span data-wd-action=next{#slides3} class=wd-place-e wd-padding wd-icon-right/span
	span data-wd-action=prev{#slides3} class=wd-place-w wd-padding wd-icon-left/span
/div
    

		
    Para concluir o presente tópico, será demonstrado como implementar ações para parar e acelerar a transição dos elementos.
    

		
    Para implementar tais ações, será utilizado o atributo data-wd-data. Esse atributo altera o valor do atributo HTML data e reage ao evento click.
    

		
    Após algumas modificações e a construção de um navegador para manipular as ações, a visualização ficará da seguinte forma:
    

		
    
			
    Slide 1
    
			
    Slide 2
    
			
    Slide 3
    
			
    Slide 4
    
		
    

		
    
			
			
			
		
    

		
    div class=wd-b wd-size-xlarge wd-align-center id=slides4 data-wd-slide=4500 
	div style=height: 150px; class=wd-bg-blue  Slide 1/div
	div style=height: 150px; class=wd-bg-red   Slide 2/div
	div style=height: 150px; class=wd-bg-green Slide 3/div
	div style=height: 150px; class=wd-bg-black Slide 4/div
/div

nav
	span data-wd-data=wdSlide{null}${#slides4} class=wd-icon-stop    /span
	span data-wd-data=wdSlide{4500}${#slides4} class=wd-icon-play    /span
	span data-wd-data=wdSlide{1000}${#slides4} class=wd-icon-forward /span
/nav
