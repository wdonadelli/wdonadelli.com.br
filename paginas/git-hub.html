O Básico do Git+Hub

Utilizando os comandos básicos do Git e GitHub

03 de dezembro de 2020

O propósito deste artigo é descrever alguns comandos básicos do Git e a integração com o GitHub.

Segundo o sítio em 03/12/2020:

Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.
Git is easy to learn and has a tiny footprint with lightning fast performance. It outclasses SCM tools like Subversion, CVS, Perforce, and ClearCase with features like cheap local branching, convenient staging areas, and multiple workflows.
Git é um sistema de controle de versão distribuído gratuito e de código aberto projetado para lidar com tudo, desde projetos pequenos a muito grandes com velocidade e eficiência.

Segundo a Wikipédia sobre o GitHub em 03/12/2020:

GitHub é uma plataforma de hospedagem de código-fonte e arquivos com controle de versão usando o Git. Ele permite que programadores, utilitários ou qualquer usuário cadastrado na plataforma contribuam em projetos privados e/ou Open Source de qualquer lugar do mundo. Este artigo não possui tudo que as aplicações oferecem, apresenta apenas uma das maneiras de utilizar suas ferramentas.

O Início

As duas primeiras coisas a se fazer para utilizar o Git e o GitHub são instalar a aplicação e criar uma conta no sítio , respectivamente.

A instalação do Git no computador depende do seu sistema operacional. Verifique se a loja de aplicativos do sistema operacional oferece o sistema para instalação ou veja as orientações em . Com o Ubuntu 20.04 LTS, basta executar como root:

sudo apt install git

Com a conta do GitHub realizada, o endereço do seu perfil será https://github.com/usuario, onde usuario é seu identificador no GitHub.

Criando o Diretório no GitHub

Para iniciar um novo projeto no GitHub é preciso criar um diretório. Caso tenha dúvida em como proceder, observe as orientações da própria aplicação. Após criado o diretório, seu endereço será https://github.com/usuario/diretorio, onde diretorio é o identificador do diretório no GitHub.

Acredito ser produtivo criar o diretório primeiro no GitHub, pois a aplicação te guiará no processo de criação além de te dar a possibilidade de criar automaticamente os arquivos README.md, LICENSE e .gitignore, muito úteis para seu projeto.

O arquivo README.md, que utiliza a marcação markdown, é o responsável por fazer a apresentação do seu projeto no GitHub. Faça uma boa apresentação para todos conhecerem o seu objetivo, a sua aplicabilidade, a linguagem utilizada, a página da aplicação, o manual... O GitHub oferece algumas ferramentas interessantes como o wiki, para construção de um guia da sua aplicação, por exemplo, e o gh-pages, para a construção de uma página para sua aplicação, dentre outras.

O arquivo LICENSE.md diz respeito à licença do seu projeto. Se quiser que alguém utilize sua ideia, é preciso dar permissão e dizer como ela ocorrerá. Procure saber um pouco a respeito.

O arquivo .gitignore é um arquivo interessante a depender da linguagem de programação escolhida. Ao rodar/compilar sua aplicação, alguns arquivos auxiliares, de log, ou que não fazer parte do escopo do projeto, podem ser criados automaticamente pelo compilador/intepretador e o .gitignore fará com que esses arquivos não sejam colocados no seu diretório GitHub, deixando-o limpo e organizado.

Clonando o Diretório com o Git

Após a criação do diretório no GitHub, é hora de cloná-lo no sua estação de trabalho, pois é nela que o desenvolvimento ocorrerá. Para isso, entra em cena o Git.

Todos os comandos aqui demonstrados devem ser executados numa interface CLI com o uso de um terminal. No caso, foi utilizado o terminal padrão do Ubuntu 20.04 LTS. Caso ache mais confortável utilizar uma interface GUI, verifique na sua loja de aplicativos.

Crie um diretório no computador para clonar seus projetos e acesse-o:

#criando o diretório
mkdir projetos/
#acessando o diretório
cd projetos/

Para efetuar a clonagem do diretório do GitHub na estação de trabalho, execute o seguinte comando, observando o identificador real do usuário e do diretório no GitHub:

git clone https://github.com/usuario/diretorio.git

Acesse o diretório clonado para iniciar os trabalhos:

cd diretorio/
